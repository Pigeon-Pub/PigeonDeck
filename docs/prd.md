# PigeonDeck 重开发 PRD

> **权威声明**: 本文件为 PigeonDeck V1 的功能需求规格。最终裁决以 `context/构想蓝图2.md` 为准——当本文件与其冲突时，以构想蓝图2 为准。

## 1. 产品目标

PigeonDeck 是一个面向网页验收、UI 修改反馈和 AI 编码交付的 Chrome/Edge MV3 浏览器扩展。它以 MarkIt 为主要行为参考，从零使用 Vite + TypeScript + Manifest V3 重建一个更精简、可维护的版本；以 ClickDeck 作为移动组件、吸附参考线和移动任务 prompt 的局部参考。

第一版目标不是复刻 MarkIt 的全部复杂度，而是做到"体验接近 MarkIt，但工作流更统一"：保留标注、区域批注、文本/样式直接编辑、复制文本、复制图片、清空、设置、教程、popup 禁用、右键菜单和 file 页面支持；去掉 Dev Mode / Prototype Mode 双模式、高级输出模板、复杂 pin 样式合集、额外导出入口和营销型视觉效果。

**架构基调**: V1 单页面工作流、单列纵向工具盘（仅图标 + hover tooltip）、pigeonlib 设计系统全套照搬、展开即激活批注模式。

## 3. 用户与核心场景

目标用户是在网页上向 AI 编码工具表达修改意图的人，包括产品验收、前端 UI 调整、页面复核和原型评审场景。

核心场景：

1. 用户打开页面，点击 logo 悬浮球展开单列纵向工具盘，**自动进入批注模式**。
2. 在批注模式下，点击页面元素弹出批注面板，写一句或多句修改说明，并可通过修改栏直接修改文本或样式形成即时预览。
3. 长按拖拽框选页面区域，生成区域批注，填写一句修改说明。
4. 点击移动按钮进入移动模式，单击选中组件，点住即拖到目标位置，默认吸附并显示参考线，按住 Alt 自由移动。
5. 点击复制文本（瞬时动作），将所有标注、编辑、区域和移动预览整理成 Codex 可执行的任务清单（默认英文）。
6. 点击复制图片（瞬时动作），生成长图叠加批注信息，复制到剪贴板或下载为文件。
7. 使用合并的撤销/重做按钮回退或恢复操作；点击清空并二次确认后恢复当前页干净状态。

## 4. 功能需求

### 4.1 Logo 悬浮球与单列纵向工具盘

#### 收起态：Logo 悬浮球

- 品牌 logo（抽象鸽子轮廓，SVG 一笔线稿风）显示在**邮政金圆形底色**上（`#b8842c`）。
- 悬浮球圆形直径 40px，白色或浅色 logo 叠加在金色圆形之上。
- 默认停靠**右下角**，距边缘 16px。
- 点击 → 展开工具盘。
- **长按 → 拖拽移动悬浮球位置**（展开后整体插件随之移动）。
- 位置持久化到 localStorage。

#### 展开态：单列纵向工具盘

展开后显示单列图标栏。Logo 进入工具栏顶部。

**按钮顺序（从上到下）**：

| 位置 | 按钮 | 行为 |
|------|------|------|
| 1 | **Logo** | 点击展开/收起；长按拖拽整体。展开时默认进入批注模式。 |
| 2 | **移动** | 进入拖拽模式。单击=选中组件；点住即拖；Alt+拖=自由移动。 |
| 3 | **复制文本** | 瞬时动作，复制后按钮内短暂反馈。不改变当前模式。 |
| 4 | **复制图片** | 瞬时动作，生成长图并复制/下载（设置中切换）。不改变当前模式。 |
| 5 | **撤销/重做** | 合并为一个按钮（左半=撤销，右半=重做）。默认 50 步上限。 |
| 6 | **清空** | 点击弹出贴工具盘的小确认弹层，确认后清除所有批注/编辑/移动/历史。 |
| 7 | **设置** | 打开设置面板，贴工具盘出现。打开时暂停页面选择和拖动。 |

**批注模式**：展开工具盘默认进入批注模式——这是工具栏的默认工作态。无需独立批注按钮：
- 展开 → 自动进入批注。工具栏无激活按钮高亮（批注是默认底态）。
- 点击移动 → 进入移动模式，移动按钮高亮（邮政金）。
- 再点移动 → 取消选中，回到默认批注态。
- 点击设置 → 进入设置模式。关闭面板后回到默认批注态。

**规则**：
- 按钮仅图标，hover 时在旁边显示 tooltip（文字标签）。
- 当前激活工具按钮高亮（邮政金），并在工具盘外层显示轻量描边。默认批注态无按钮高亮。
- 展开方向：默认向右下；靠近视口底部 → 向上展开；靠近视口右边 → 向左展开。
- 收起工具盘 → 退出激活模式，但**不丢失已保存的批注**。

### 4.2 元素选择与粒度切换

- 支持两级选择粒度：智能视觉组件块、精确 DOM 元素。
- 默认粒度可在设置中选择。
- 选中框附近提供紧凑的粒度切换按钮。
- 支持用户自定义快捷键快速切换粒度。
- 如果点击位置无法形成有效目标，显示工具栏轻提示并保持当前工具模式。

### 4.3 标注与直接编辑

- 批注模式下，点击元素后在目标附近显示批注浮层。
- 标注核心模型是 `target + instruction`。
- 批注浮层内集成样式/内容编辑，不拆成独立工具。

#### 修改栏智能切换

修改栏按元素类型自动切换显示的控件：

| 元素类型 | 修改栏内容 |
|----------|-----------|
| 文本元素 | 文字内容编辑框 + 排版控件（字体/字号/字重/颜色/对齐/行高/字距/margin/padding） |
| 图片 | 尺寸/边框/圆角 + 替换图片按钮 |
| 视频 | 尺寸/边框/圆角 + 替换视频按钮 |
| 按钮/容器 | 外观控件（边框/圆角/阴影/不透明度）+ 排版/间距控件 |
| 其他 | 根据当前 CSS 属性动态列出最相关控件 |

修改栏常驻显示，不折叠。

#### 高级样式区

左侧垂直导航，4 个分类按使用频率排列：

| # | 分类 | 包含控件 |
|---|------|----------|
| 1 | **排版** | 字体、字号、字重、颜色、对齐方式、行高、字距、margin、padding、列表样式 |
| 2 | **尺寸** | width、height、min/max 尺寸、display 模式切换 |
| 3 | **外观** | 背景色、背景图、边框、圆角、阴影、透明度、替换图片/视频 |
| 4 | **调试** | 完整 computed style、DOM 信息（tagName、className、ID、属性列表） |

**双入口规则**：已在修改栏常驻区出现过的控件（如颜色），在高级区的「排版」和「外观」中再次出现——方便用户在高级区集中操作。

**控件行为**：
- 所有修改即时预览，进入撤销/重做历史。
- **调色盘**：收起 = 小色块 + 色值文本；展开 = 完整取色器 + **局部取色推荐**（从当前元素及祖先链取 computed color/background-color，去重后按出现频率排序，取前 7）+ 透明度滑杆。
- **数值控件**：文本框 + 右侧 `+`/`-` 按钮，同时支持原生 input[type=number] 滚轮微调。

#### 样式修改输出格式

每个修改输出为「人类指令 + CSS 属性前后值表」：

```
- 修改按钮颜色：蓝色 → 金色
  | background-color | #2563eb | #b8842c |
```

### 4.4 区域批注

- 长按 ≥300ms → 拖拽框选后创建区域批注对象。
- 实时显示品牌金边框 + 浅金填充矩形预览。
- 松手后弹出区域批注面板（用户填写修改说明）。
- 内部记录：视口位置 + 文档位置（相对 body）+ **框内可见元素列表**（用于复制文本中 AI 精准定位）+ 坐标范围（用于复制图片叠加矩形）。
- 区域批注不立即要求选择导出文本或图片；导出由工具栏的复制文本/复制图片按钮决定。
- 区域批注和元素标注共用同一套编号系统。

### 4.5 移动组件

- 移动模式下，单击选中组件。
- **点住即拖**：默认无阈值（可在设置中开启防误触阈值）。移动真实元素显示预览；**自动吸附到视口内其他块级元素**（水平中心对齐、垂直间距对齐、边缘对齐），显示参考线。
- **参考线**：虚线，颜色根据页面背景自动切换白/黑，显示对齐方位名称（如「水平居中」「8px 间距」）。
- **Alt+拖拽**：隐藏参考线、取消吸附，记录为 free move。
- **松手**：自动生成移动任务。不弹出确认卡片。
- 同一个组件多次移动时，复制文本只输出"初始位置 -> 最终位置"；撤销/重做历史仍保留每一步移动。
- **吸附算法**：扫描当前视口内所有可见块级元素 → 计算拖拽元素与其的水平/垂直距离 → 阈值内（默认 4px）显示参考线并自动吸附到位 → 实时更新参考线位置。

### 4.6 复制文本

- 复制文本输出为任务清单格式，**默认语言为英文**（设置中可切换中文或跟随界面）。
- 输出包含页面上下文、全局编辑规则、操作列表和每个操作的定位信息。
- 操作类型至少包括：标注、区域批注、样式修改、移动组件。
- 上下文保持精简：包含目标选择器、文本摘要、位置、必要 computed style，不默认包含完整 HTML/CSS。
- 移动任务必须说明 Source、Target、初始位置、最终位置、吸附参考线或 free move 状态。
- 输出应明确提示 AI：视觉坐标是定位线索，不应直接变成脆弱的 `top/left` 硬编码；优先使用现有布局、flex/grid、gap、margin、order 或局部 wrapper。
- **同元素多操作合并**：若同一元素同时有批注 + 样式修改 + 移动，输出为一条合并操作（Type: `Annotation + Style Modification + Move`），所有信息聚合在同一编号下。
- **去重规则**：同一元素多处修改自动去重合并为最终状态；去重维度为元素标识（选择器）+ 修改类型。

### 4.7 复制图片

- 复制图片只复制图片，不额外复制文本。
- V1 范围：**单页长图**，覆盖当前页面所有被标注内容（基于标注元素的文档坐标计算范围）。
- 叠加当前页面截图、批注编号、连线、区域框、移动预览。
- 设置中可切换「复制到剪贴板」或「下载为文件」。
- 是否在图片角落显示 URL/时间等元数据由设置开关控制。
- 复制失败时显示轻提示并保留所有状态，不自动改为复制文本。

### 4.8 清空

- 清空按钮第一次点击后，在工具盘旁出现小确认弹层。
- 小确认弹层包含确认清空和取消动作。
- 确认后清除当前页的标注、区域、直接编辑、移动预览、撤销/重做历史。
- 清空后编号从 1 重新开始。
- **不设自动清空**：所有清空由用户手动触发，或浏览器 tab 关闭时资源自然释放。

## 5. 状态与数据规则

- 暂存生命周期按标签页会话实现。
- 同一 tab 刷新同一完整 URL 时自动恢复状态。
- 关闭 tab 后清理该 tab 会话状态。
- 页面键使用完整 URL。
- 如果刷新后找不到原移动组件，跳过移动预览并轻提示；移动任务记录保留，复制文本仍包含该意图。
- 如果刷新后找不到原编辑元素，跳过编辑预览并轻提示；编辑任务记录保留，复制文本仍包含该意图。
- **标注编号**：按标注创建时间递增分配；**删除不重排**（编号一旦分配即固定）；仅清空后从 1 重置。
- **移动任务合并**：同一组件多次移动，复制文本只输出初始→最终位置；撤销历史保留每一步。
- **复制文本去重与多操作合并**：同一元素多次修改 → 自动去重合并为最终状态（去重维度：元素选择器 + 修改类型）。同一元素同时有批注+样式修改+移动 → 输出合并为一条（Type: `Annotation + Style Modification + Move`），不拆成两条。
- 撤销/重做覆盖全部编辑操作：标注、区域、直接编辑、移动预览、清空。默认 50 步上限，设置中可调（最高 9999 步）。

## 8. 验收标准

### 8.1 功能闭环

- 用户可以通过悬浮球完成标注、区域批注、直接编辑、移动组件、复制文本、复制图片、撤销/重做、清空和设置。
- popup 可以完成全局禁用、站点禁用、禁用列表管理和 file 权限提示。
- 右键菜单可以快速进入标注流程。
- file 页面可以正常使用核心功能。
- PDF 页面显示明确的不支持提示。

### 8.2 输出质量

- 复制文本能把所有操作组织成 Codex 可执行的任务清单（默认英文）。
- 同一组件多次移动时，输出只保留初始位置到最终位置。
- 同元素多操作（标注+样式修改+移动）合并为一条输出。
- 样式修改包含用户意图和属性前后值。
- 复制图片只产生图片，不附带文本。
- 长图能看清批注编号、连线、区域框和移动预览。

### 8.3 状态恢复

- 同一 tab 刷新同一完整 URL 后，能自动恢复可定位的标注、编辑和移动预览。
- 找不到目标时不乱改页面，显示轻提示，并保留任务记录。
- 清空后页面状态、历史和编号全部重置。

### 8.4 Git 安全

- 后续实现前必须确保 `reference-projects/` 不进入新的提交。
- 如果 `reference-projects/` 已在索引中，执行阶段应仅移出索引并保留本地文件。

### 8.5 测试与验收环境

- 使用 vitest 做单元测试（状态管理、数据模型、导出逻辑、选区/粒度计算、吸附算法）。
- 使用 Playwright 做 E2E 测试，加载本地 HTML 夹具页面（按钮、卡片、表单、图片、嵌套布局、可移动组件）。
- E2E 覆盖：完整标注→编辑→区域框选→移动→复制文本→复制图片→撤销→清空→状态恢复→设置面板交互。
- **file:// 协议验证**：E2E 加载本地夹具页面时，验证 file:// 权限下全链路可用。
- 使用至少一个真实网站在 Chrome/Edge 上手动冒烟测试。
- `npm test` 全部通过后方可合并；核心用户路径 E2E 全绿。

## 10. 开发方法论

### 10.1 自驱动开发架构

采用"放养式"开发模式：UI 规范通过构想蓝图 2 打磨完成后，后续实现全部由 AI 自主驱动。需产出：

- **项目规范与约束文档**：技术栈（Vite + TypeScript + MV3）、代码风格、目录结构、提交规范。
- **面向目标优化的提示词**：一个可直接启动、无需人工干预的自主推进提示词，使 AI 在目标文档驱动下逐个功能完成实现。
- **Git 提交策略**：每完成一个细分功能提交一次。主 session 管理 worktree 及合并；子代理在隔离环境中执行实现任务。

### 10.2 测试方法

- **单元测试**：vitest，测试文件与源码同目录（`*.test.ts`）。覆盖状态管理、数据模型（标注/移动/编辑/撤销重做）、导出逻辑（去重/合并/格式化）、选区与粒度计算、吸附算法。
- **E2E 测试**：Playwright，加载本地 HTML 夹具页面。场景覆盖：完整标注→编辑→区域框选→移动→复制文本→复制图片→撤销→清空→状态恢复→设置面板交互。
- **手动冒烟**：至少一个真实网站在 Chrome/Edge 上加载扩展完成全链路。
- **测试夹具**：本地固定 HTML 夹具覆盖按钮组、卡片列表、表单控件、图片网格、嵌套 flex 布局、绝对定位元素。夹具与源码放在同一仓库（如 `tests/fixtures/`）。

### 10.3 规范文档维护纪律

- **颗粒化规范**：每条规则一个文件 + `docs/conventions/INDEX.md` 索引。
- **杜绝冗余**：删除过时规则，避免描述重叠。
- **变更必同步**：新增或更新规范时，必须同步更新索引文件。

## 11. 工程形态

### 11.1 技术栈

| 层 | 技术 | 备注 |
|----|------|------|
| 框架 | Vite + TypeScript | 多入口构建：background + content |
| 平台 | Chrome/Edge Manifest V3 | MV3 标准 |
| 测试 | Vitest（单测）+ Playwright（E2E） | 同目录 `*.test.ts` |
| 代码风格 | `strict`、`noUnusedLocals`、`noUnusedParameters` | module resolution = `Bundler` |
| 国际化 | `default_locale` + `__MSG_*__` | 中/英双语 |
| 静态资源 | `public/`，经 `web_accessible_resources` 暴露 | manifest.json 放 public/ |

构建命令：
```bash
npm install
npm run build       # vite build → dist/
npm run dev         # vite build --watch
npm run typecheck   # tsc --noEmit
npm test            # vitest run src
npm run e2e         # playwright test
npm run i18n:check  # validate language files
```

### 11.2 OpenDesign 兼容策略

OpenDesign 是本地优先的开源桌面设计工具，PigeonDeck 作为 Chrome MV3 扩展，兼容策略分两个层面：
