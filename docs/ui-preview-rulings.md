# UI 预览裁决记录

> **定位**：PigeonDeck V1 的 11 轮 UI 预览迭代裁决历史。产品决策已全部整合到 [context/构想蓝图2.md](../context/构想蓝图2.md) 和 [docs/v1-plan.md](v1-plan.md)。本文件仅保留 UI 控件/布局层面的裁决细节供实施参考。
> **注意**：本文件中提到的外部参考项目名仅为历史记录，不是产品源码构成。

## UI 预览草稿与初期视觉方向（2026-06-27 会话裁决）

**交付物**：`preview/` 组件画廊。`preview/pigeonlib.css`（设计令牌 + 基础控件配方）+ `preview/index.html`（画廊宿主，复刻 `ui-extract/components.html` 形式）+ `preview/parts/01–16-*.html`（16 个 PD 表面，各自渲染在中性网页背景上）。验证方式：在 `preview/` 起静态服务后用浏览器/Playwright 截图逐卡核对。

**裁决 1 — 初期视觉方向：参考布局 + 邮政金皮肤**
- 初期先做标准浏览器扩展工具盘布局（上下圆形 pill 胶囊工具盘、圆形悬浮球、面板/控件比例、控件行/分段/开关/左导航等结构），配色用已裁决的 pigeonlib（暖纸白 + 邮政金 + 宋体标题 + Lucide）。
- **PD 自己的特色留到后期**再做。
- 玻璃拟态仍属构想蓝图2 禁区，**不**沿用玻璃质感。
- V1 确认不进预览：开发/原型双模式、多输出模板、输出详细度、多 pin 样式、Markdown 导出、隐藏标注。

**裁决 2 — 批注/编辑面板去顶栏**
- 删掉批注面板中冗余的顶部栏（拖拽条 + 标题 + 模板按钮 + 关闭 ✕）——「这里没有任何有用信息」。
- 面板**无显式关闭按钮**：靠 Esc / 点击面板外 / 保存关闭。
- 需要显示的信息与操作集成到**底栏**：左 `位号 · 元素类型 · 位置(px)`，右 删除 + 保存。该底栏模式与批注卡片底栏统一（卡片底栏右侧为 删除 + 修改）。

**裁决 3 — 第二轮收紧（2026-06-27 截图批注）**
总原则：**这是悬浮工具，占用最小视线成本**——全局更紧凑、减少「廉价」的横线分割。
- **悬浮球/工具盘**：悬浮球尺寸 **42px**；工具盘列宽与悬浮球一致（按钮 34px border-box + padding 3 + 边 1 = 42）。
- **工具盘按钮顺序**：撤销/重做移到 **Logo 与「移动」之间**，做成**横向药丸**（左半撤销 / 右半重做）；**Logo 展开态也用金底浅羽**（与收起球一致）；**设置图标改齿轮**；**去掉 pill 内的短横线分割**。
- **批注/编辑面板**：去掉顶部「希望 AI 怎么改」**标题**（标题无信息量）。~~去掉 prose 输入框~~ → 见裁决 4 修正：**文本框保留**，只去标题。
- **高级样式区**：① 加**底框**（导航 + 控件收进一个浅底容器，内部调色盘去内边框避免卡片套卡片）；② **删除**底部 Before/After 总览；③ 左侧分类导航在该类有改动时显示**变更角标**（左竖条 + 计数点）；④ 变更说明改为**贴在小节标题右侧**、精简为「旧 → 新」（如 `圆角 7px → 12px`）。
- **设置面板**：加**分类导航**（通用/交互/输出/帮助，**圆形选中样式**、更窄更紧凑）；**限定长度**（按分类收敛，不再一长条）；**去掉逐行横线**，靠留白 + 分组标题分隔。
- **区域批注**：删除按钮改**垃圾桶图标**（与面板/卡片底栏统一）。
- 预览复核：用无依赖 CDP 脚本（Node 24 全局 WebSocket 驱动 `chrome-headless-shell`）量测各表面真实高度并截图，回填 `index.html` 的 iframe 高度。

**裁决 4 — 第三轮（2026-06-27 截图批注，先提交首版再改）**
- **先提交**：改动前先把首版预览提交一次（提交信息**不提及参考项目名**），再迭代。
- **悬浮球/Logo 尺寸锁定**：收起态悬浮球 **42px 不变**；展开后 Logo 金底球也**保持 42px、不变大也不缩小**；工具盘其余按钮放大到 38px，整体**适当加宽**。
- **撤销/重做药丸**：箭头太淡 → **加深（正文色）+ 加粗（stroke 1.9）**，合并药丸放大到 38×24。
- **批注文本框保留**：只去掉「希望 AI 怎么改」标题，**文本框本身保留**，占位提示改为 **`补充修改说明…`**（主态/高级态都保留）。
- **面板内减少圆角矩形**：**单行控件统一改药丸形**（输入框 / 下拉 / 数字步进 / 分段 / 按钮 / 图标按钮 / 色值条 / 粒度切换）；**多行文本框与分组容器**（modbar、高级样式底框）**保留圆角矩形**，避免全药丸失去层次。
- **数字步进顺序**：改为 `输入 → 步进(+/−) → 单位`（加减放单位前面）。
- **有图标的分段用图标**：如对齐 左/居中/右 改用对齐图标（Lucide align-left/center/right）。
- **清空确认弹层修复**：原弹层工具盘仍是旧版（旧顺序 + 短横线 + 旧设置图标），且箭头未对准清空键 → 同步为新工具盘布局，清空按钮置危险态，弹层箭头重新对准清空键。

**裁决 5 — 多状态补全 + 右键菜单 + 暗色 + 缩放句柄（2026-06-27 第四轮）**
本轮把组件画廊从「每个多状态表面只画一态」扩成「每个状态/子页各自成卡」，并补齐蓝图里还没画的表面。新增 `preview/parts/17–32-*.html`，`pigeonlib.css` 增 `.pd-menu`/`.pd-menu-item`（上下文菜单）与 `.pd-range`（滑杆）两个基元。
- **设置 4 子页各自成卡**：通用 / 交互 / 输出 / 帮助（原 13 保留为交互页）。**帮助页新增「检查更新」行**（用户要求）。**导出语言**确认为多语言分段「英文 / 中文 / 跟随」、**默认英文**（用户要求）。
- **高级样式 4 子页各自成卡**：排版 / 尺寸 / 外观 / 调试（原 12 保留为外观页）。调试页 = 只读 DOM 信息 + computed style，computed 区 `.pd-scroll` 限高、按行整齐截断。
- **新表面**：调色盘展开（取色器 + 局部取色推荐 7 色 + 透明度滑杆）、双击文本内联编辑、图片/视频替换流程、语言选择+搜索（输入「中」高亮命中）。
- **右键菜单（仅这两处；§14 的「工具按钮右键快捷设置」属 V2，不做）**：位号圆右键「修改批注 / 删除批注」（删除项火漆红）、浏览器右键嵌入「用 PigeonDeck 快速标注」（金底高亮 + Alt+P，与原生项拉开）。
- **页面级轻提示族**：复制成功(橄榄绿√) / 保存成功 / 找不到目标元素(中性金) / PDF 不支持(火漆红)。
- **暗色版关键面板**：批注主面板 / 高级样式 / 设置 各出一张 `data-theme="dark"` 卡，dark host 底，验证双主题。
- **选中框四角方块 = 可缩放句柄（方向变更）**：用户裁决移动模式下**拖四角直接改尺寸**，不再只在面板里调 width/height。**这超出原蓝图2「改尺寸只在面板调」的范围**，已同步更新蓝图2 §4.3。预览中选中框四角方块视觉本就在，正好对应此语义。**确认归属 V1**（用户澄清「v1 就要」，不推 V2）。
- **截断修复纪律**：逐卡 CDP 截图复核，凡 `scrolling=no` 下被切的（替换弹层、浏览器右键菜单、帮助页、`.num-sm` 的「px」单位等）回填高度/宽度直至不切。
- **粒度（元素/容器）的作用**（解释，已记入认知）：网页 DOM 深层嵌套，单击一处可能命中很深的节点。PigeonDeck 默认选「智能组件块」并给「元素↔容器」切换，让用户**不必精确点中目标 DOM 节点**即可选对范围；移动/缩放/批注的 Target 选择器随之确定。
- **待补（本轮内，待用户裁决后落地）**：富文本内联编辑增强（见裁决 6）。

**裁决 6 — 逐字符富文本内联编辑（2026-06-27 第五轮）**
- 诉求：双击文本进入编辑后，**选中其中几个字**即可单独改格式（参考 Word/Docs 的选区浮条），不止整段统一改。
- 形态裁决：**双行全功能浮条**，贴在选区上方。第一行 字体▾ / 字号▾ / 字色(A+色条) / 高亮▾；第二行 B I U S / 上标 x² 下标 x₂ / 对齐▾ / 列表。
- 语义：浮条对**选中字符 run**生效（逐字符富文本），与修改栏的元素级排版**并行**——浮条管选区、修改栏管整个文本元素。改动即时预览、进撤销历史。
- 落地：预览 `preview/parts/24-inline-edit.html` 改为该双行浮条 + 选区 run（「今日截止」放大+加粗+品牌金）；蓝图2 §4.2、§5.1 已补「内联富文本浮条」规格。

**裁决 7 — 中文化 + 调色盘重做 + 批注主态补元素类型（2026-06-27 第六轮）**
- **高级样式中文化**（蓝图硬约束：UI 文案默认简体中文）：尺寸页 width/height/min-w/max-w/display → 宽度/高度/最小宽/最大宽/显示；排版页 margin/padding → 外边距/内边距；调试页表头加「计算样式」（computed 原始属性名属只读 readout，保留英文）。
- **display 用图标**：尺寸页 display 分段改图标段（块级=横行 / 弹性=纵列 / 网格=网格 / 行内=绕排，icon-only + 中文 tooltip），diff 用中文「块级 → 弹性」。
- **调色盘重做** `23-palette.html`：① 去掉 hex 行冗余「92%」（透明度交给下方滑杆，用户「不要」）；② 修复饱和度方块**左上缺块**（`overflow:hidden` + 边框改 `--cbd` 加强）；③ 7 个局部取色块**铺满卡片宽度**（`flex:1` + `aspect-ratio:1`，实现「卡宽 = 七色宽」）；④ 新增 **RGB 三通道微调**（R/G/B 输入，与 hex 并存）。
- **批注面板主态补元素类型**（落实蓝图2 §5.1 修改栏智能切换）：新增图片（替换图片 + 宽/高/圆角/边框）、按钮/容器（背景/圆角/边框/阴影/不透明度/外内边距）、**陌生元素自适配**（未匹配固定类型时按 computed style 自动列出最相关控件，带「自动」角标 + 说明条）。原 11 标注为「文本」主态。`preview/parts/33–35`。

**裁决 8 — 颜色控件统一 + 修改栏统一风格 + 调试翻译 + 全面图标化/自检（2026-06-27 第七轮）**
- **统一颜色控件 `.pd-color`**：所有用色处一律「**色块方 → 色值药丸 → 取色器**」一行三件，点色块展开调色盘（=调色盘卡 23）。替换 11/12/20/30/31/34/35 的 picker-top/colorctl，并删除内联的局部取色推荐/透明度（归入展开态调色盘），移除孤立 CSS。用户确认「这样才对」。
- **修改栏统一为高级样式风格**：modbar → 底框 `.modbox` + `.prop`/`.prop-h`（标题左 + 变更说明右 + 控件在下），与高级样式视觉一致（11/30/33/34/35）。副作用：陌生元素页标签随之变深变粗，解决「看不清」。
- **调试页特殊化**：默认**全英文**（DOM Info / Computed Style）；左导航在「调试」下方增**翻译图标**；点击 → 中文（标签名/类名/计算样式…）。默认卡 `22-adv-debug.html`（英文）+ 译后卡 `36-adv-debug-zh.html`（中文）。
- **该用图标处补图标**：主题（日/月）、复制图片方式（剪贴板/下载）、显示（块级/弹性/网格/行内，含陌生元素页）、对齐（既有）。
- **修 33 图片替换**：去掉孤立缩略图，改为单按钮 + 图标（「替换图片（本地 / URL）」），消除「怪怪的、风格不一致」。
- **遮挡修复**：位号圆右键菜单改为**从位号圆向下弹出**（箭头指回卡片），不再遮挡被标注卡片，演示完整。
- **中英混用清理**：排版字体「系统 Sans-serif → 系统默认」；调试 readout 的英文为**有意**（默认英文 + 翻译开关），非混用。
- **四角缩放句柄**：用户澄清「v1 就要」→ 确认归属 **V1**（非 V2）。

**裁决 9 — 调用式组件化 + 双入口单源 + 输出预览 + 审计修复（2026-06-27 第八轮）**
用产品经理 + 资深前端视角审计后重构。
- **自定义元素组件层** `preview/pigeon-components.js`：原生 Web Components，渲染到 **light DOM**（复用全局 pigeonlib.css），各 part 用 `<pd-…>` 调用。元素：`<pd-panel/modbox/advbox/prop/foot/adv-nav/set-nav/color/num/range/seg-display/seg-align/seg-deco>`。图标内联在组件里（单源）。面板外壳样式（panel/pbody/pfoot/modbox/prop/advbox/设置壳…）**提取进 pigeonlib.css 单一真相源**，各 part 只留宽度。iframe 加 `allow-scripts`。
- **双入口单源属性原件**（落实蓝图2 §5.1 双入口规则）：属性控件注册表 `FIELDS`，修改栏与高级样式都用 `<pd-prop field="bgColor">` 引用**同一原件**——改一次两处同步。修改栏=按元素类型挑的高频子集，必为高级样式全集真子集。
- **透明棋盘格**：`.pd-color .sw` 加棋盘格底 + `.fill` 叠色，alpha 颜色（如阴影色）透出，亮/暗均验证。
- **高级样式取材丰富**：排版+文字装饰(B/I/U/S)/列表/大小写；尺寸+object-fit/overflow/min·max；外观+背景图/边框色/阴影色/滤镜模糊；间距=外内边距。**刻意排除 absolute position / top / left / z-index**（蓝图硬约束「勿硬编码 top/left，优先 flex/grid/gap」）。
- **输出预览两卡（核心交付物可视化，原先零可视化）**：`37-output-text` 复制文本任务清单（§7.1 结构 + 前后值表 + 勿硬编码 top/left 提示，默认 English + 复制按钮）；`38-output-image` 复制图片单页长图（缩略叠加位号圆/连线/区域框/移动幽灵 + 剪贴板/下载）。
- **定向修复**：位号圆右键菜单改**向上弹出 + 箭头指回**（不遮卡片）；调试页翻译图标后右侧滚动区**扩高填满**（不留空）；陌生元素 display 改 4 图标段（消除空槽）；设置行数值控件统一 118px 宽。
- **批注主态补元素类型** 与高级样式共用原件：文本(⊂排版)/图片(⊂尺寸·外观)/按钮容器(⊂外观·间距)/陌生(自动选取同一原件)。
- 验证：无依赖 CDP 逐卡截图，组件化后像素无回退；棋盘格透明可见；pin 上弹不遮卡；调试不留空；输出两卡完整；暗色无残留。同步蓝图2 §5.1（丰富控件 + 排除绝对定位）。



**裁决 10 — 组件层缺陷修复 + 八向句柄 + 粒度入设置（2026-06-27 第九轮）**
本轮针对截图反馈，修组件化后暴露的问题，并落实几处交互裁决：
- **自定义元素空盒**：`<pd-prop>` 等自定义元素默认 `display:inline`，inline 宿主含块级子节点 → 左缘渲染出空药丸（用户称「两个垃圾内容」）。修法：pigeonlib 给 `pd-panel/pd-modbox/pd-advbox/pd-prop/pd-foot` 等显式 `display:block`，`comp()` 跳过纯空白文本节点再 append。
- **变更说明左贴标题**：`.prop-h` 由 `space-between` 改 `flex-start`，diff 紧贴标题（如「字号 6→12」），不再甩到最右。
- **选中框八向句柄**：`.pd-selbox` 四角 + 四边中点共 8 个句柄（05、25 同步），蓝图2 §4.3 更新。
- **粒度移入设置、去浮窗**：移动选中不再显示浮动「元素/容器」切换；粒度由设置「默认选择粒度」决定（用户裁决）。蓝图2 §4.3 更新。
- **对齐加两端对齐**：`pd-seg-align` 增第 4 项 justify（两端对齐图标）。
- **颜色控件恒为整行**：所有颜色统一「色块-色值-取色器」一行（不塞进 grid2 半格挤压）；透明色显示棋盘格底（pd-color 已内置）。外观页边框色/阴影色改整行。
- **尺寸页中英混用/排版炸**：去掉 objectFit（image 专属 + cover/contain 英文），溢出改全宽中文段（可见/隐藏/滚动）。
- **位号圆右键菜单上弹**：从位号圆向上弹 + 底部箭头指回，不再遮卡片（26 重做）。
- **调试翻译 + 滚动扩展**：调试导航在「调试」下加「翻译」项，默认英文、点后转中文（22 英文态 / 36 中文态）；computed 滚动区扩高填满，翻译项下不留空。
- **背景图巨图标**：`.pd-btn svg` 限 15px，修背景图按钮图标撑大。
- 验证：无依赖 CDP 逐卡截图复核；面板逐卡量真实高度回填 index（排版 738 / 外观 694 等防截断）。

**裁决 11 — 控件紧凑化：数值「大胶囊套小圆胶囊」+ 胶囊统一 + 颜色块圆形 + 下拉小圆球箭头（2026-06-28 第十轮）**
本轮按截图反馈把「占用最小视线成本」落到控件级，统一收紧所有控件。
- **单行胶囊高度统一 ~30px**：input / select / 数值 / 颜色（色块·色值条）垂直 padding `9px→6px`，与分段（对齐/溢出）等高。**多行文本框**（批注内容、文字内容）保留圆角矩形、不参与压缩。
- **数值控件「大胶囊套小胶囊」**（用户裁决，纠正了「两个独立并排胶囊」的误解）：单个大胶囊内含 `值 → 单位px → 右侧嵌套小圆胶囊[＋−]`。小胶囊**纯色底（`--chv`）无阴影**，`＋−` 用 Lucide plus/minus **图标**、互相靠近 1px，单位 px 贴小胶囊左侧。比「独立两胶囊」更省宽，能放进高级样式带导航的窄两列。**坑**：`<button>` 默认 UA 内边距(~12px)叠加图标(14px>文字9px)使小胶囊变宽 → 重置 `padding:0` + `min-width:20` + icon 13px 收回原宽（CDP 实测 55px→43px）。
- **文字内容自适应高度**：从单行 input 改 `textarea` + `field-sizing:content`，换行自动增高。
- **胶囊内文字 + ＋− 颜色 = 「隐藏」色 `--ctx2`**：input/select/数值/单位/色值/＋− 统一用分段未选项（如溢出·隐藏）的中性灰；多行文本框文本仍用深色 `--ctx` 保证可读。
- **所有颜色块圆形**：`.pd-color .sw` / `.pd-swatch` / 调色盘 `.chip`·`.s` 一律 `border-radius:50%`；透明 alpha 色的棋盘格随之裁成圆形。
- **下拉箭头用小圆球包住**：去原生箭头（`appearance:none`），右侧叠一个**与小胶囊同样式的 `--chv` 小圆球** + chevron（`--ctx2`）。落地为单源组件 `<pd-select value>`（comp 渲染 `select + .pd-sel-arrow`），FIELDS 5 处 + 设置语言选择器统一引用；inline-edit 富文本浮条(24)的紧凑下拉属独立紧凑表面，本轮不并入。
- 验证：CDP 逐卡复核 11/12/20/21/17/30/31 + 暗色；小胶囊实测回到 ~43px；颜色圆/棋盘格圆/下拉小圆球在亮暗均一致；面板高度实测仍 ≤ index 回填值（无截断）。
