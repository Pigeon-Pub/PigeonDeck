# PigeonDeck V1 实施计划

> **状态**：已完成的 PigeonDeck V1 实施与验收基线。本文用于查阅既有模块、阶段决策和验收范围，不再是当前任务计划。
> **前置阅读**：[产品规格](../product/product-spec.md) — 产品定位、设计系统、交互规格、输出格式的完整定义。本文侧重于**实施层面**的内容：开发阶段划分、验收标准、文件模块清单。

---

## 1. 产品概述

PigeonDeck 是面向网页验收、UI 修改反馈和 AI 编码交付的 Chrome/Edge MV3 浏览器扩展。用户在任意网页上标注、修改元素、移动组件预览，然后一键"复制文本"（生成 Codex/AI 可执行的任务清单）或"复制图片"（单页长图叠加批注）。

### 1.1 设计哲学

PigeonDeck 的 UI 目标是"安静、温柔、雅致、可靠"——有一点宣纸的气息但不走古风。关键词：宣纸感、温润留白、淡雅简朴、清简专业。

整体判断标准：如果某个视觉元素看起来"很努力"，大概率就不该要。UI 应该让人觉得：**轻、静、准、有气质、不吵、但很好用**。

### 1.2 技术栈

| 层 | 技术 | 备注 |
|----|------|------|
| 框架 | Vite + TypeScript | 多入口构建：background + content |
| 平台 | Chrome/Edge Manifest V3 | MV3 标准 |
| 测试 | Vitest（单测）+ Playwright（E2E） | 同目录 `*.test.ts` |
| 代码风格 | `strict`、`noUnusedLocals`、`noUnusedParameters` | module resolution = `Bundler` |
| 国际化 | `default_locale` + `__MSG_*__` | 中/英双语 |
| 静态资源 | `public/`，经 `web_accessible_resources` 暴露 | manifest.json 放 public/ |

### 1.3 顶层架构决策

| 决策 | 结论 |
|------|------|
| 多页面工作流 | V1 只做单页面。数据预留扩展点，V2 做多页统一导出 |
| 工具盘布局 | 单列纵向（仅图标，hover 出 tooltip）。点击 Logo 展开/收起，长按 Logo 拖拽整体位置 |
| 设计系统 | pigeonlib 全套照搬（邮政金 `#b8842c` 点睛色、Fraunces+思源宋体标题、Lucide Icons） |
| 位号系统 | 删除不重排。编号一旦分配就固定，仅清空后从 1 重置 |
| 展开即激活 | 展开工具盘 → 自动进入批注模式（无独立批注按钮、无按钮高亮） |
| 导出语言 | 可设置，默认英文 |
| OpenDesign 兼容 | 硬约束 — 输出格式兼容 OpenDesign 的 prompt 约定，选型避免不兼容依赖 |
| 主题 | 亮/暗双主题，V1 完整实现 |

### 1.4 V1/V2 边界

| 功能 | V1 | V2 |
|------|-----|-----|
| 多页面统一导出 | ❌（数据预留扩展点） | ✅ |
| 长图（单页） | ✅ | — |
| 多页多图 | ❌ | ✅ |
| 工具按钮右键快捷设置 | ❌ | ✅ |
| 暗色主题 | ✅ | — |
| 多页面清空（三次确认） | ❌ | ✅ |
| PDF 支持 | ❌（仅提示不支持） | 待评估 |

## 2. 实施阶段

### 阶段 1：工程骨架（预计 3-5 天）

**目标**：可加载的 MV3 扩展 + Shadow DOM 宿主 + 设计令牌。

- [ ] Vite + TypeScript + MV3 项目搭建
- [ ] `manifest.json` 配置（`default_locale`、`web_accessible_resources`、权限、**`update_url`**）
- [ ] 多入口构建：`background`（service worker）+ `content`
- [ ] **分发与自动更新**：主渠道上架 Chrome 网上应用店 + Edge 加载项（浏览器原生自动更新）；兜底自托管 `.crx` + `updates.xml`（GitHub Releases），manifest 写 `update_url` 指向之（裁决12 #2）
- [ ] Shadow DOM 宿主注入 + 四层结构（Control/Panel/Overlay/Feedback）
- [ ] pigeonlib 设计令牌移植（CSS 变量：色彩/字体/圆角/阴影/动效；含 `interpolate-size:allow-keywords` 面板长度动画）
- [ ] 亮/暗双主题 via `data-theme` 属性
- [ ] Lucide Icons 集成（内联 SVG）
- [ ] 基础 i18n 框架（`_locales/en/` + `_locales/zh_CN/`）
- [ ] logger 基础设施

**验证**：
- `npm run build` 产出 `dist/` 可加载扩展
- `npm run dev` watch 模式正常
- `npm run typecheck` 零错误
- 加载到 Chrome 后 Shadow DOM 注入、设计令牌生效

- [ ] 数值控件（大胶囊套小圆胶囊 + −）
- [ ] 面板长度变化柔和动画（`interpolate-size` px→auto 过渡，切分区/展开高级样式；裁决12 #5）

**批注卡片结构**：
- 上半部分：完整批注文本（有批注时显示）
- 下半部分：`调整项：原值 → 新值`（有修改时显示）
- 底部栏：位号·元素类型·位置(px) | 删除+修改按钮

**验证**：
- 单击文本/图片/按钮/容器元素均弹出正确类型的修改栏
- 样式修改即时预览、撤销可回退

**目标**：生成 Codex/AI 可执行的任务清单，默认英文。

- [ ] 输出结构：[Page Context] + [Global Editing Rules] + [Operations]

**验证**：

### 阶段 15：测试（贯穿，预计 3-5 天）

**目标**：单测 + E2E + 手动冒烟。

#### 单元测试 (vitest)
- 状态管理：标注/编辑/移动/撤销重做的增删改查
- 数据模型：标注编号分配、删除不重排、移动任务合并
- 导出逻辑：去重/合并/格式化/输出结构
- 选区与粒度计算
- 吸附算法（距离计算、阈值匹配、参考线生成）

#### E2E 测试 (Playwright)
- 加载本地 HTML 夹具页面（按钮/卡片/表单/图片/嵌套 flex/绝对定位）
- 覆盖场景：完整标注→编辑→区域框选→移动→复制文本→复制图片→撤销→清空→状态恢复→设置面板交互
- **file:// 协议验证**：E2E 加载本地夹具页面验证 file:// 权限下全链路

#### 手动冒烟
- 至少一个真实网站在 Chrome/Edge 上加载扩展完成标注/编辑/移动/复制文本/复制图片全链路
- Popup、右键菜单、file 页面路径独立验证

## 4. 验收标准

### 4.1 功能闭环

- [ ] 用户可通过悬浮球完成标注、区域批注、直接编辑、移动组件、复制文本、复制图片、撤销/重做、清空和设置
- [ ] Popup 可完成全局禁用、站点禁用、禁用列表管理和 file 权限提示
- [ ] 右键菜单可快速进入标注流程
- [ ] file:// 页面可正常使用核心功能
- [ ] PDF 页面显示明确的不支持提示

### 4.2 输出质量

- [ ] 复制文本能把所有操作组织成 Codex 可执行的任务清单（默认英文）
- [ ] 同一组件多次移动时输出只保留初始→最终位置
- [ ] 同元素多操作（标注+样式修改+移动）合并为一条输出
- [ ] 样式修改包含人类指令和属性前后值
- [ ] 复制图片只产生图片不附带文本
- [ ] 长图能看清批注编号、连线、区域框和移动预览

### 4.3 状态恢复

- [ ] 同一 tab 刷新同一完整 URL 后自动恢复可定位的标注/编辑/移动预览
- [ ] 找不到目标时不乱改页面，显示轻提示，保留任务记录
- [ ] 清空后页面状态、历史和编号全部重置

### 4.4 测试门禁

- [ ] `npm test` 全部通过（vitest 单测）
- [ ] 核心用户路径 E2E 全绿（Playwright）
- [ ] `npm run typecheck` 零错误
- [ ] `npm run i18n:check` 通过

### 4.5 Git 安全

- [ ] 仓库不包含任何参考项目源码
- [ ] `.gitignore` 正确覆盖 `node_modules/`、`dist/`、日志文件

## 5. 开发纪律

### 5.1 Git 提交策略
- 每完成一个细分功能提交一次
- 主 session 管理 worktree 及合并
- 子代理在隔离环境中执行实现任务
- 提交信息用中文，一句话说清做了什么

### 5.2 规范维护
- 颗粒化规范：每条规则一个文件 + `docs/conventions/README.md` 索引
- 杜绝冗余：删除过时规则，避免描述重叠
- 变更必同步：新增或更新规范时同步更新索引文件

### 5.3 OpenDesign 兼容
- 复制文本输出格式兼容 OpenDesign 的 prompt 约定
- 选型避免引入与 OpenDesign 生态不兼容的依赖
