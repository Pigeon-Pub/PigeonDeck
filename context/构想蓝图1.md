# 构想蓝图 1 — 优化后的提示词集

---

## 1. 项目初始化与自驱动架构

### 1.1 环境初始化

为 PigeonDeck 项目安装 `ponytail` skill。

### 1.2 自驱动开发架构

采用"放养式"开发模式：UI 规范打磨完成后，后续实现全部由 AI 自主驱动。需要产出以下配套文档：

- **明确的项目规范与约束文档**：定义技术栈（Vite + TypeScript + MV3）、代码风格、目录结构、提交规范等硬性约束。
- **面向 `/goal` 优化的目标提示词**：一个可直接启动、无需人工干预的自主推进提示词，使 AI 在目标文档驱动下逐个功能、逐个细分功能地完成实现。
- **Git 提交策略**：每完成一个细分功能即提交一次，commit message 遵循规范化格式。主 session 管理 worktree 和合并，子代理在隔离环境中执行任务。
- **规范文档维护纪律**：颗粒化规范（每条规则一个文件 + INDEX.md 索引），杜绝冗余，新增/变更规范时必须同步索引。

## 2. Agent Routing（写入 CLAUDE.md）

```markdown
## Agent Routing

- **主 session 职责**：任务拆分、规划、协调、审查、管理 git worktree 与合并。不直接执行实现任务。
- **子代理委派**：将所有实现任务委派给子代理。大多数任务使用 background subagent 并行执行；仅当任务足够小且与当前上下文高度耦合时，才在主线程直接执行。
- **子代理生命周期**：探索 → 规划 → 实现 → 检查 → 测试，循环直至任务完成，然后交由主 session 审查合并。任务足够简单时可省略中间过程。
- **隔离策略**：每个任务在独立 git worktree 中运行，完成后再合并到主分支。
```

## 3. 多语言架构

项目从第一天起原生支持多语言（i18n）。要求：

- 使用 `default_locale` + `__MSG_*__` 国际化方案（MV3 标准）。
- 设置面板中提供语言下拉选择器，支持搜索过滤。
- 所有 UI 文案抽取为语言资源文件，不得硬编码。

## 5. 性能与流畅性

**核心要求：快如闪电、轻量化、流畅。**

- 内容脚本注入体积要小，按需懒加载模块。
- DOM 操作最小化，避免不必要的 reflow/repaint。
- 动画使用 GPU 加速属性（transform、opacity），禁止触发 layout 的动画属性。

---

## 6. 动画一致性

**所有动画的速度曲线和时长必须高度统一**，营造舒适、一致的高级感。

- 定义统一的动画 token：时长（如 150ms/250ms/400ms 三档）、缓动曲线（如 ease-out）。
- 全局复用，禁止组件内私自定义时长或曲线。
- 展开/收起/浮入/淡入等同级动画使用同一档位。

## 7. 元素描述功能（可开关）

扩展激活后（工具菜单展开时），鼠标 hover 任意页面控件，在其描边高亮框上方显示一个简洁的元素类型标识（如 `div`、`span`、`img`）。

- 设置中提供开关。
- 标识样式轻量：小字号、半透明背景、跟随 hover 目标实时移动。

## 8. 多页面批注与统一输出

### 8.1 多页面批注流程

支持跨页面连续批注，所有页面的修改统一缓存，最终一键导出完整提示词。

**缓存机制：**

- 批注完成后无需手动保存。只要用户未点击"清空"，该页所有改动（批注、样式修改、元素编辑、移动预览）持久缓存。
- 关闭页面后重新打开同一 URL，自动恢复缓存中的批注状态。
- 切换到其他页面继续批注，每个页面独立缓存。
- 尝试拦截页面导航/刷新事件，降低批注缓存因页面变动而失效的风险。

**导出规则：**

- 点击"复制提示词"时，导出所有已缓存页面的全部改动。
- 提示词按页面分节输出，格式为结构清晰的英文。
- 遵循最小化信息原则：精简但不模糊，结构化但不冗余，给出必要约束条件。
- 提示词需去重（同一元素多次修改合并为最终状态）。

### 8.2 位号系统

**位号**是跨页面的全局标注编号，按标注先后顺序递增分配。

- 页面 A 标注了 1、2 → 切换到页面 B，位号从 3 开始。
- 删除某个批注后，后续位号自动重排（如 1、2、3(删)、4 → 变为 1、2、3）。
- 位号在页面内和导出提示词中保持一致。

## 9. 标注元素的视觉标记与交互

### 9.1 标注框与位号

每个被批注/修改的元素周围显示圆角矩形边框。边框左上角附一个圆形标记，内含位号数字。

### 9.2 展开/收起批注卡片

- **点击位号圆形**：展开或收起该元素的批注内容卡片。
- **默认状态**：收起（可在设置中改为默认展开）。
- **右键位号圆形**：弹出上下文菜单，提供"删除批注"和"修改批注"选项。

### 9.3 卡片布局规则

展开的批注卡片为紧凑矩形框，结构如下：

| 区域 | 内容 | 条件 |
|------|------|------|
| 上半部分 | 完整批注文本 | 仅有批注时显示 |
| 下半部分 | `调整项：原值 → 新值`，每项一行 | 仅有修改时显示 |
| 底部栏 | 左对齐：`位号 · 元素类型 · 元素位置(px)`；右对齐：🗑 删除按钮 + ✏ 修改按钮 | 始终显示（窄窄一条） |

- 文本框自适应内容高度，完整展示不截断。
- 面积最小化：无上半部分则不显示，无下半部分则不显示，只保留有内容的区域。
- 标注面板的底部栏布局相同，右对齐改为**展开按钮**（展开/收起面板内容）。

**定位规则：**

- 优先紧邻标注元素放置，智能选择上下左右方向。
- 展开的卡片之间必须保持间距，防止视觉混淆。
- 可遮挡未标注元素，但不可遮挡其他已标注元素（含其边框和位号和批注卡片）。
- 若所有邻近位置都会遮挡已标注元素 → 移动到最近的合适位置，绘制虚线箭头（可弯曲）从卡片连接回位号圆形。

## 10. 面板格式

### 10.1 工具栏布局

采用**单列纵向布局**，仅显示图标、无文字。功能名称鼠标悬停时显示 tooltip。

### 10.2 右键快捷设置

每个工具按键支持右键打开一个紧凑的功能专属设置小面板（设置项与总设置页面共享同一份数据源和逻辑，不拆成多个难维护的文件）。设置面板本身除外（右键无小面板）。

---

## 11. 工具按钮顺序与功能

从上到下依次为：

### 11.1 Logo（展开/收起）

点击展开或收起工具栏。展开后默认进入**批注模式**。Logo 先用占位符。

### 11.2 批注模式

- **单击**：选中元素，弹出标注面板。
- **双击文本元素**：进入文字编辑。
- **双击图片/视频**：进入替换流程。
- **双击产生的修改**同样显示标注框。
- **左键长按**：进入区域框选模式，拖出矩形标注区域（仅标注，不支持编辑，不吸附对齐）。

  **待定项（需进一步讨论）：**
  - 长按触发阈值：多久算"长按"？与单击/双击如何区分、如何规避冲突？
  - 框选过程中的视觉反馈：拖拽时的实时矩形预览样式？
  - 框选完成后的定位锁定：如何保证框选区域在页面内容变动/滚动后不移位？
  - 框选区域的用途定义：框选后能做什么？仅标注说明？还是后续可关联到具体元素？
  - 如何向 AI 表示框选结果：框选区域应导出为什么格式（元素选择器 + 坐标范围？CSS 片段？截图区域描述？），使 AI 能理解"用户框了什么"。

### 11.3 拖拽模式

点击进入拖拽模式（按钮高亮标识）。

- **单击左键**：选中元素。
- **长按左键拖动**：移动元素。默认为块级对齐吸附。
- **Alt + 拖动**：自由移动到页面任意位置，不吸附。
- 拖拽模式下自动显示块级元素对齐参考线（虚线，颜色根据页面背景自动切换黑/白）。

### 11.4 复制提示词

点击后将所有页面的全部改动和批注生成并复制为结构化提示词。

- 多页面批注场景下需读取缓存，确保完整。
- 提示词去重、精简、结构清晰。

### 11.5 复制标注图片

将当前视口生成标注长图并复制。

- 长页面自动截断，但图片必须完整覆盖所有被标注内容。
- 基于当前页面分辨率生成。
- 被标注内容在图片中按规则展开显示。
- 多页面场景下生成多张图（需提前缓存和渲染）。
- 设置中可选"复制到剪贴板"或"下载为文件"。

### 11.6 撤销 / 重做（合并为一个按钮）

- 撤销和重做共用一个按键的空间（左右或上下分区）。
- 默认支持 50 步，可设置最高 9999 步。

### 11.7 清空

清除当前页面（或所有页面）的全部批注和修改。

- **单页面清空**：需二次确认（点击 → 弹出确认面板 → 再次点击确认）。
- **多页面批注清空**：需三次确认（点击 → 弹出确认面板 → 输入确认文字/点击确认 → 最终执行）。
- 多页面清空的确认面板需明确提示影响范围（将清除 N 个页面的全部批注）。

### 11.8 设置

打开全局设置面板。右键无小面板。

## 12. 标注面板详情

### 12.1 面板结构

默认弹出紧凑小面板，无任何非必要描述文字。

| 区域 | 说明 |
|------|------|
| 上半部分 | 标注内容文本框（自适应文本量调整大小） |
| 下半部分 | 修改栏（根据选中元素类型智能调整内容） |

### 12.2 修改栏 — 文本元素

**直接展示**（最高频操作）：字体、字号、颜色、对齐方式等。

**折叠区域**：不常用文本功能折叠在"显示更多"按钮中，点击后向下展开全部编辑功能。

**智能分类**：文本相关的所有可用功能自动归入"文本"分类栏，分类名称随选中元素类型动态变化（如选中图片则变为"图片"）。其他功能按类别分组排列。已在"文本"栏中出现的功能，仍需在其所属分类栏中再次出现（双入口）。

### 12.3 修改栏 — 非文本元素

根据元素类型（图片、视频、按钮等）智能显示对应的功能控件，分类排列规则同上。

### 12.4 调色盘

- **收起状态**：一个小色块显示当前颜色 + 右侧文本框显示色值。
- **展开状态**（点击色块触发）：
  - 完整取色器。
  - 下方 7 个推荐颜色，算法：读取页面所有颜色 → 按"该颜色在页面中出现次数"与"该颜色在当前控件类型中出现次数"加权排序。
  - 支持透明度调节。

### 12.5 数值控制

明确为增减型的参数（如字号），在文本框右侧添加 `+` / `-` 按钮。

### 12.6 功能参考范围

目标功能覆盖全面，但应基于本项目设计系统重新实现。

## 13. 智能防截断

**所有悬浮 UI 元素（工具栏、批注卡片、设置面板、上下文菜单、tooltip 等）必须智能避让页面边缘，防止被截断。**

- 检测元素与视口边界的距离，超出时自动翻转方向。
- 工具栏示例：悬浮球靠近页面底部时，展开方向变为**向上**，但除 Logo 仍保持在最下方外，其余工具的排列顺序不变。
- 每个面板/弹出类组件均需独立计算边界约束。

---

## 14. 悬浮元素跟随

**标注框、位号标记、批注卡片等所有叠加在被标注元素上的 UI，必须跟随被标注元素的实时位置移动。**

- 当页面滚动、窗口 resize、或被标注元素因 DOM 变动发生位移时，叠加 UI 需实时更新位置——PigeonDeck 必须做到。
